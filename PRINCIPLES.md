## 🎯 프로젝트 정체성

### 프로젝트명
**집결정 (Jip01)** - "집 결정을 도와주는 앱"

### 핵심 미션
30대 여성 직장인의 **"판단의 외로움"**을 해소한다.

### 핵심 가치 제안
```
문제: "집 보러 갈 때 뭘 봐야 할지 모르겠어요 😢"
해결: "켜놓고 걸으면 전문가가 속삭여요"

→ GPS 기반 유튜브 임장 가이드 (Hands-free)
→ 판단을 대신하지 않고, 판단의 기준을 제공
```

### 타겟
- **1차**: 30대 여성 직장인, 6개월 내 이사 예정
- **2차**: 부동산 초보, 첫 집 구매/전월세
- **지역**: 서울 및 수도권

### 핵심 미디어
- **YouTube 링크** (MP3 파일 사용 금지)
- 이유:
  1. 앱 용량 최소화
  2. 즉시 업데이트 가능
  3. 기존 유튜브 채널 활용
  4. 시각 정보 추가 가능
  5. SEO 및 무료 마케팅

## 🚫 3대 절대 원칙

이 원칙들은 **절대 타협 불가**하며, 모든 기능과 디자인 결정의 기준이 된다.

### 원칙 1: 설명하지 마라, 느끼게 하라

**배경**: 
사람들은 설명을 읽지 않는다. 특히 바쁜 30대 직장인은 더욱 그렇다.

**적용**:
```
❌ BAD: "이 앱은 GPS 기반으로 위치를 인식하여 해당 지역의 임장 
         가이드를 자동으로 재생합니다..."

✅ GOOD: [큰 버튼 1개]
         "지금 여기서 임장 시작하기"
         (한 줄 부가 설명) "켜놓고 걸으면 전문가가 속삭여요"

→ 15초 안에 가치 전달
→ 3초 안에 첫 행동 유도
→ 설명 = 실패
```

**이론적 근거**:
- Effort Discounting: 노력이 필요하면 가치가 할인됨
- Cognitive Load Theory: 인지 부하 최소화
- Flow Theory: 즉각적인 피드백

**측정 방법**:
- 첫 사용자의 80% 이상이 15초 내에 버튼 클릭
- 온보딩 화면 0개
- 튜토리얼 0개

---

### 원칙 2: 판단을 요구하지 마라, 안심시켜라

**배경**:
"판단의 외로움"은 선택지가 많아서가 아니라, 선택의 결과를 확신하지 못해서 발생한다.

**적용**:
```
❌ BAD: "코스를 선택하세요"
         [초급] [중급] [고급]
         → 사용자 불안: "나는 어떤 레벨이지?"

✅ GOOD: "지금 계신 곳은 성수동이에요"
         "5분 코스 자동으로 시작할게요"
         → 사용자 안심: "앱이 알아서 해주네"

❌ BAD: "이 동네 괜찮을까요?"
         → 판단 요구

✅ GOOD: "이 동네에서 체크할 3가지"
         "✓ 지하철역 7분 (기준: 10분 이내)"
         "✓ 대형마트 있음 (기준: 15분 이내)"
         "? 초등학교 재개발 여부 확인 필요"
         → 판단 기준 제공
```

**이론적 근거**:
- Cognitive Offloading: 인지 부담을 외부로 전가
- Epistemic Satisfaction: 인식론적 만족감
- Scaffolding: 비계 제공
- Self-Determination Theory: 자율성 지원

**측정 방법**:
- 앱 내 선택지: 3개 이하
- 자동 감지/추천 비율: 80% 이상
- "뭘 해야 하나요?" 문의: 0건

---

### 원칙 3: 흐름을 깨지 마라, 유지하라

**배경**:
몰입은 가장 강력한 사용자 경험이다. 한 번 깨지면 다시 회복하기 어렵다.

**적용**:
```
❌ BAD: 
   [영상 재생 중]
   → 팝업: "로그인하시겠어요?"
   → 사용자: 이탈

✅ GOOD:
   [영상 재생 중]
   → 아무것도 하지 않음
   → 영상 완료 후:
      "좋았어요! 다음 동네도 들어보시겠어요?"

❌ BAD: 로딩 3초 이상
   → 사용자: "느리네" → 이탈

✅ GOOD: 
   → YouTube 스트리밍 (즉시 시작)
   → 로딩 표시 최소화
```

**이론적 근거**:
- Flow Theory (Csikszentmihalyi): 몰입 상태 유지
- 손실 회피 (Loss Aversion): 중단 = 손실
- Zeigarnik Effect: 미완성 과제의 심리적 압박

**측정 방법**:
- 영상 중 이탈률: 5% 이하
- 평균 시청 완료율: 80% 이상
- 재생 중 팝업/알림: 0개

## 👤 타겟 사용자

### 핵심 페르소나: "지수" (30세, 직장인)

**인구통계**:
- 나이: 30-39세
- 성별: 여성
- 직업: 사무직 (연봉 4,000-7,000만원)
- 거주: 서울 월세, 부모님과 분리 거주
- 상태: 6개월 내 이사 예정 (결혼/독립)

**심리상태**:
- 현재 감정: **압도됨, 외로움, 불안**
- 고민: "잘못 선택하면 어떡하지?"
- 욕구: "누군가 확신을 줬으면 좋겠어요"
- 기대: "전문가처럼 볼 수 있으면..."

**행동 패턴**:
- 평일: 출퇴근으로 바쁨
- 주말: 집 보러 다님 (하루 3-5곳)
- 정보: 유튜브, 네이버 카페, 친구 조언
- 결정: 혼자서 판단하기 어려움

**부동산 숙련도**:
- 초보: 전문 용어 모름
- 경험: 월세 1-2회, 매매 0회
- 지식: 유튜브로 학습 중
- 네트워크: 부동산 지인 없음

**디지털 습관**:
- 스마트폰: 하루 4-5시간
- 앱: 네이버, 카카오, 인스타, 유튜브
- 선호: 간단, 직관적, 예쁜 디자인
- 거부: 복잡, 어려운, 투박한 UI

## 🔍 문제 정의 및 해결 전략

### 핵심 문제: "판단의 외로움"

**문제 분석**:
```
상황: 집 보러 갔을 때
  ↓
"뭘 봐야 하지?"
"이게 중요한 건가?"
"다른 집도 이런가?"
"이 가격이 적정한가?"
  ↓
혼자 판단해야 함
  ↓
불안 → 결정 미루기 → 기회 상실
```

**기존 솔루션의 문제**:
1. **유튜브 영상**: 집에서만 봄, 현장에서 못 봄
2. **부동산 앱**: 데이터만, 판단 기준 없음
3. **지인 조언**: 타이밍 안 맞음, 전문성 부족
4. **공인중개사**: 신뢰 문제, 수수료 부담

**우리 솔루션**:
```
GPS 감지
  ↓
"지금 계신 곳은 성수동이에요"
  ↓
YouTube 영상 자동 재생
  ↓
전문가 음성 가이드
  ↓
"지금 왼쪽 보세요..."
"이 건물이 중요한 이유는..."
"체크할 3가지: ..."
  ↓
사용자: "아, 이렇게 보는구나!"
  ↓
안심 → 확신 → 결정
```

**차별화 요소**:
1. **현장 중심**: 집 보러 갈 때 사용
2. **Hands-free**: 걸으면서 들음
3. **판단 기준 제공**: "이래서 좋아요" (확신)
4. **전문가 목소리**: 730번 임장 경험
5. **부담 없음**: 무료 시작, 5분 짧은 길이

## 🛡️ 실패 회피 전략

Perplexity가 분석한 7가지 앱 실패 함정과 회피 전략:

### 1. 완벽 추구 (Perfectionism)

**함정**:
```
"다크 모드도 만들고..."
"20가지 색상 변형..."
"AR 기능도 넣고..."
→ 출시 못 함
```

**회피**:
```
✅ MVP 먼저
✅ 기능 1개만 완벽
✅ 빠른 출시 > 완벽한 출시
```

---

### 2. 기능 과부하 (Feature Bloat)

**함정**:
```
"AI 챗봇도..."
"커뮤니티도..."
"게임화도..."
→ 복잡도 폭발
```

**회피**:
```
✅ 버튼 1개 원칙
✅ 새 기능 = 기존 기능 삭제
✅ "No" 말하는 연습
```

---

### 3. 타겟 불명확 (Target Confusion)

**함정**:
```
"남성도, 여성도..."
"20대도, 40대도..."
"서울도, 지방도..."
→ 누구에게도 안 맞음
```

**회피**:
```
✅ 30대 여성만
✅ 페르소나 "지수" 고정
✅ 다른 타겟 = 다른 앱
```

---

### 4. 기술 중심 (Tech-Driven)

**함정**:
```
"최신 기술 써보자"
"AR 멋있어"
"AI 넣으면 트렌디"
→ 사용자 불필요
```

**회피**:
```
✅ 문제 중심
✅ 사용자 중심
✅ 기술은 도구일 뿐
```

---

### 5. 데이터 무시 (Ignoring Data)

**함정**:
```
"내 생각엔..."
"사람들은 당연히..."
"난 이게 좋은데..."
→ 주관적 판단
```

**회피**:
```
✅ 10명 테스트
✅ 재듣기율 측정
✅ 데이터 기반 결정
✅ A/B 테스트
```

---

### 6. 마케팅 소홀 (No Marketing)

**함정**:
```
"좋은 앱이면 알아서 퍼질 거야"
→ 아무도 모름
```

**회피**:
```
✅ HOC 커뮤니티 활용
✅ 유튜브 채널 연동
✅ SEO (YouTube)
✅ 인플루언서 협업
```

---

### 7. 수익화 늦음 (Delayed Monetization)

**함정**:
```
"일단 사용자 모으고..."
"나중에 수익화..."
→ 지속 불가능
```

**회피**:
```
✅ 4,900원 프리미엄 (초기부터)
✅ 매몰 비용 활용 (3개 동네 후)
✅ 명확한 가치 제공
```

## 📚 이론적 배경

우리 앱은 다음의 이론과 연구를 기반으로 한다:

### 행동경제학

**1. 손실 회피 (Loss Aversion)**
```
적용: "3개 동네 수집했어요!" (진행도)
효과: 중단 = 손실 느낌 → 계속 사용
```

**2. 매몰 비용 효과 (Sunk Cost)**
```
적용: 3개 코스 완료 후 프리미엄 제안
효과: "여기까지 했는데..." → 전환율 증가
```

**3. 기준점 효과 (Anchoring)**
```
적용: "지하철역 7분 (기준: 10분)"
효과: 판단 기준 제공 → 확신 증가
```

---

### 인지심리학

**1. 인지 부하 이론 (Cognitive Load)**
```
r = -0.83 (강한 역상관)
적용: 설명 최소화, 버튼 1개
효과: 인지 부하 ↓ → 사용성 ↑
```

**2. 플로우 이론 (Flow Theory)**
```
r = 0.74 (강한 상관)
적용: 영상 재생 중 방해 금지
효과: 몰입 → 완료율 증가
```

**3. 자기결정성 이론 (Self-Determination)**
```
적용: 자율성 지원 (강요 X, 제안 O)
효과: 내재적 동기 → 재사용
```

---

### 사용자 경험 (UX)

**1. Scaffolding (비계 제공)**
```
적용: "지금 왼쪽 보세요..."
효과: 단계별 안내 → 성공 경험
```

**2. Epistemic Satisfaction (인식론적 만족)**
```
적용: "이래서 좋아요" (이유 설명)
효과: 불안 해소 → 확신
```

**3. Cognitive Offloading (인지 외주화)**
```
적용: "체크할 3가지" (판단 기준)
효과: 부담 감소 → 행동 증가
```

---

### 습관 형성

**1. Hooked Model (니르 이얄)**
```
Trigger: 집 보러 갈 때
Action: 버튼 1개 클릭
Reward: 전문가 안내 (가변 보상)
Investment: 완료 코스 저장

→ 습관 형성 루프
```

**2. Tiny Habits (BJ Fogg)**
```
B = MAP
Behavior = Motivation × Ability × Prompt

적용:
- Motivation: 판단의 외로움 (높음)
- Ability: 버튼 1개 (쉬움)
- Prompt: GPS 자동 감지

→ 행동 발생 확률 최대화
```

---

### 데이터 기반 의사결정

**100만 번 시뮬레이션 결과**:
```
1. 도파민 경제학: r=0.87
   → 즉각 보상 (영상 시작) 중요

2. 사회적 비교: r=0.92
   → "3개 동네 수집!" 공유 효과

3. 손실 회피: 강력
   → 진행도 표시 필수

4. 플로우 이론: r=0.74
   → 몰입 유지 = 성공

5. 인지 부하: r=-0.83
   → 단순함 = 성공
```
